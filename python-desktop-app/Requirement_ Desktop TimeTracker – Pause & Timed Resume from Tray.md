**Requirement: Desktop TimeTracker – Pause & Timed Resume from Tray**

| ID | Requirement | Description | Priority | Notes / UX Behaviour |
| ----- | ----- | ----- | ----- | ----- |
| DT-PAUSE-01 | Pause tracking from tray icon | The desktop TimeTracker must allow the user to pause tracking directly from the system tray/menu bar icon via a single click or tray context menu option. | Must-have | \- Tray context menu includes an entry **“Pause Tracking”** when tracking is active. \- When paused, the menu label toggles to **“Resume Tracking”**. \- Tray icon state changes (e.g., overlay or color) to clearly show **Paused** vs **Tracking**. |
| DT-PAUSE-02 | Timed pause with auto-resume | When the user chooses to pause, the app must offer an option to pause for a fixed duration, after which tracking resumes automatically. | Must-have | \- On selecting **Pause**, show a small dialog/popover from the tray with preset durations (e.g., 5, 10, 15, 30, 60 minutes) and an option **“Until I manually resume”**. \- User selects a duration and confirms; tracking stops immediately and a countdown timer starts. |
| DT-PAUSE-03 | Timer display for paused state | While paused with a timed pause, the user must be able to see the remaining pause time. | Should-have | \- Tray icon tooltip shows **“Paused – resumes in 07:32”** (mm:ss). \- Optional: clicking the tray icon opens a small status window showing the remaining time and a **“Resume now”** button. |
| DT-PAUSE-04 | Auto-resume behaviour | When the pause timer ends, tracking must resume automatically without additional user input. | Must-have | \- At timer expiry, tracking state switches back to **Active**. \- Tray icon returns to **Tracking** state. \- If the app is minimized or in background, auto-resume still occurs. |
| DT-PAUSE-05 | Resume nudge / notification | When tracking resumes automatically, the app must show a small, non-intrusive notification to the user. | Must-have | \- Show a desktop notification/toast: **“Time tracking resumed”** with app name and icon. \- Notification is clickable and brings the app to front (configurable). \- If the OS does not support notifications, show an equivalent in-app banner on next focus. |
| DT-PAUSE-06 | Manual resume overrides timer | If the user resumes tracking manually before the timer ends, the timer must be cancelled. | Must-have | \- Selecting **“Resume Tracking”** from tray while paused immediately cancels the scheduled auto-resume and starts tracking. \- Any pending notification for auto-resume must not fire. |
| DT-PAUSE-07 | Persistence across app restarts | The app must handle restarts/crashes gracefully while a timed pause is active. | Should-have | \- Pause state and remaining time are stored locally (e.g., in encrypted session/settings). \- On restart, if the scheduled resume time is in the future, app restores **Paused** with updated remaining time; if in the past, app starts **Tracking** and shows “Tracking resumed while app was restarted” message. |
| DT-PAUSE-08 | Default values & settings | The user must be able to configure default pause durations and whether auto-resume is enabled. | Nice-to-have | \- Settings screen: list of allowed pause durations \+ toggle for **“Offer timed pause options”** and **“Show resume notification”**. \- Default config: timed pause enabled, typical durations (5/15/30/60 min), notifications on. |
| DT-PAUSE-09 | Privacy & analytics handling | Paused periods must not be recorded as active work or sent to backend. | Must-have | \- No activities/screenshots/logs must be captured while paused. \- Timesheet and analytics must treat paused intervals as non-working time. |
