## What is PKCE?

PKCE (pronounced "pixy") is an extension to OAuth 2.0 that adds an additional layer of security to the authorization code flow. It was originally designed for mobile and native applications but is now recommended for all OAuth clients per RFC 9700 (2025).

### Key Components

| Component | Description |
|-----------|-------------|
| `code_verifier` | A cryptographically random string (43-128 characters) generated by the client |
| `code_challenge` | A transformed version of the verifier: `BASE64URL(SHA256(code_verifier))` |
| `code_challenge_method` | The transformation method used (we use `S256` for SHA-256) |

---

## Why PKCE is Required

### The Vulnerability (Without PKCE)

In the standard OAuth authorization code flow, a malicious application on the same device could potentially:

1. Register to handle the redirect URI (`http://localhost:51777/auth/callback`)
2. Intercept the authorization code before the legitimate app receives it
3. Exchange the stolen code for access tokens

### How PKCE Prevents This

With PKCE, even if an attacker intercepts the authorization code, they cannot exchange it for tokens because they don't have the `code_verifier` (which never leaves the legitimate client).

```
Attacker has: authorization_code ✓
Attacker needs: code_verifier ✗ (only exists in legitimate app's memory)
Result: Token exchange fails
```

### Compliance Requirements

Per **RFC 9700 (January 2025)** - "Best Current Practice for OAuth 2.0 Security":

> "Authorization servers MUST support PKCE. Clients MUST use PKCE code challenge methods that do not expose the PKCE verifier in the authorization request. Currently, S256 is the only such method."

## Code Changes

### Desktop App (`desktop_app.py`)

#### 1. `get_auth_url()` Method

**Location:** Lines 542-578

**Before:**
```python
def get_auth_url(self):
    state = secrets.token_urlsafe(32)
    self.tokens['oauth_state'] = state
    self._save_tokens()

    params = {
        'audience': 'api.atlassian.com',
        'client_id': self.client_id,
        'scope': 'read:me read:jira-work write:jira-work offline_access',
        'redirect_uri': self.redirect_uri,
        'state': state,
        'response_type': 'code',
        'prompt': 'consent'
    }
```

**After:**
```python
def get_auth_url(self):
    # Generate state for CSRF protection
    state = secrets.token_urlsafe(32)

    # PKCE: Generate code_verifier (43-128 characters, URL-safe)
    code_verifier = secrets.token_urlsafe(64)

    # PKCE: Create code_challenge = BASE64URL(SHA256(code_verifier))
    code_challenge = base64.urlsafe_b64encode(
        hashlib.sha256(code_verifier.encode()).digest()
    ).decode().rstrip('=')

    # Store state and code_verifier for callback verification
    self.tokens['oauth_state'] = state
    self.tokens['code_verifier'] = code_verifier
    self._save_tokens()

    params = {
        'audience': 'api.atlassian.com',
        'client_id': self.client_id,
        'scope': 'read:me read:jira-work write:jira-work offline_access',
        'redirect_uri': self.redirect_uri,
        'state': state,
        'response_type': 'code',
        'prompt': 'consent',
        'code_challenge': code_challenge,          # NEW
        'code_challenge_method': 'S256'            # NEW
    }
```

#### 2. `handle_callback()` Method

**Location:** Lines 580-603

**Before:**
```python
def handle_callback(self, code, state):
    stored_state = self.tokens.get('oauth_state')
    if state != stored_state:
        raise ValueError("Invalid state parameter - possible CSRF attack")

    response = requests.post(
        f"{self.ai_server_url}/api/auth/atlassian/callback",
        json={
            'code': code,
            'redirect_uri': self.redirect_uri
        },
        ...
    )
```

**After:**
```python
def handle_callback(self, code, state):
    stored_state = self.tokens.get('oauth_state')
    if state != stored_state:
        raise ValueError("Invalid state parameter - possible CSRF attack")

    # PKCE: Get the code_verifier we stored during get_auth_url()
    code_verifier = self.tokens.get('code_verifier')
    if not code_verifier:
        raise ValueError("Missing code_verifier - PKCE flow was not properly initiated")

    response = requests.post(
        f"{self.ai_server_url}/api/auth/atlassian/callback",
        json={
            'code': code,
            'redirect_uri': self.redirect_uri,
            'code_verifier': code_verifier  # NEW: Send verifier to AI server
        },
        ...
    )
```

### AI Server (`auth-controller.js`)

#### `atlassianCallback` Function

**Location:** Lines 29-83

**Before:**
```javascript
exports.atlassianCallback = async (req, res) => {
  const { code, redirect_uri } = req.body;

  // ... validation ...

  const tokenResponse = await axios.post(
    ATLASSIAN_TOKEN_URL,
    {
      grant_type: 'authorization_code',
      client_id: clientId,
      client_secret: clientSecret,
      code: code,
      redirect_uri: redirect_uri
    },
    ...
  );
```

**After:**
```javascript
exports.atlassianCallback = async (req, res) => {
  const { code, redirect_uri, code_verifier } = req.body;  // NEW: Accept code_verifier

  // ... validation ...

  // Build token request payload
  const tokenRequestPayload = {
    grant_type: 'authorization_code',
    client_id: clientId,
    client_secret: clientSecret,
    code: code,
    redirect_uri: redirect_uri
  };

  // PKCE: Include code_verifier if provided
  if (code_verifier) {
    tokenRequestPayload.code_verifier = code_verifier;  // NEW
    logger.info('[Auth] Exchanging OAuth code for tokens (with PKCE)');
  } else {
    logger.info('[Auth] Exchanging OAuth code for tokens (without PKCE - legacy flow)');
  }

  const tokenResponse = await axios.post(
    ATLASSIAN_TOKEN_URL,
    tokenRequestPayload,
    ...
  );
```

## Flow Diagram

### Complete PKCE OAuth Flow

```
┌─────────────────┐                    ┌─────────────────┐                    ┌─────────────────┐
│   Desktop App   │                    │    AI Server    │                    │    Atlassian    │
└────────┬────────┘                    └────────┬────────┘                    └────────┬────────┘
         │                                      │                                      │
         │ 1. Generate PKCE values:             │                                      │
         │    code_verifier = random(64)        │                                      │
         │    code_challenge = SHA256(verifier) │                                      │
         │                                      │                                      │
         │ 2. Store code_verifier locally       │                                      │
         │                                      │                                      │
         │ 3. Redirect user to Atlassian ───────┼──────────────────────────────────────►
         │    (with code_challenge)             │                                      │
         │                                      │                                      │
         │                                      │                    4. User logs in   │
         │                                      │                       Atlassian      │
         │                                      │                       stores         │
         │                                      │                       code_challenge │
         │                                      │                                      │
         │◄─────────────────────────────────────┼────── 5. Redirect with auth code ────│
         │                                      │                                      │
         │ 6. Send to AI Server: ──────────────►│                                      │
         │    - code                            │                                      │
         │    - code_verifier                   │                                      │
         │                                      │                                      │
         │                                      │ 7. Forward to Atlassian: ───────────►│
         │                                      │    - code                            │
         │                                      │    - code_verifier                   │
         │                                      │    - client_secret                   │
         │                                      │                                      │
         │                                      │                    8. Atlassian      │
         │                                      │                       verifies:      │
         │                                      │                       SHA256(verifier)
         │                                      │                       == challenge?  │
         │                                      │                                      │
         │                                      │◄───── 9. Return tokens (verified) ───│
         │                                      │                                      │
         │◄──────── 10. Return tokens ──────────│                                      │
         │                                      │                                      │
         ▼                                      ▼                                      ▼
```

### Security Verification

At step 8, Atlassian performs the following verification:

```
SHA256(code_verifier) == stored_code_challenge
```

If an attacker intercepted the authorization code at step 5, they would fail at step 7 because they don't have the `code_verifier`.

## Testing

### Manual Testing Steps

1. **Restart the AI server:**
   ```bash
   cd ai-server
   npm run dev
   ```

2. **Restart the desktop app:**
   ```bash
   cd python-desktop-app
   python desktop_app.py
   ```

3. **Logout** (if currently logged in):
   - Click the system tray icon
   - Select the user menu and logout

4. **Login again** and verify the console output:

   **Desktop App Console:**
   ```
   [OK] PKCE code_challenge generated (S256)
   [INFO] Exchanging OAuth code via AI Server (with PKCE)...
   [OK] OAuth tokens received via AI Server
   ```

   **AI Server Console:**
   ```
   [Auth] Exchanging OAuth code for tokens (with PKCE)
   [Auth] Successfully exchanged code for tokens
   ```

### Verification Checklist

- [ ] Desktop app generates `code_verifier` (64 characters)
- [ ] Desktop app creates `code_challenge` using SHA256
- [ ] Authorization URL includes `code_challenge` and `code_challenge_method=S256`
- [ ] Desktop app sends `code_verifier` to AI server
- [ ] AI server includes `code_verifier` in Atlassian token request
- [ ] Token exchange succeeds
- [ ] User is successfully authenticated

## Backward Compatibility

The implementation maintains backward compatibility with older desktop app versions:

| Desktop App Version | Behavior |
|---------------------|----------|
| **New (with PKCE)** | Sends `code_verifier`, AI server includes it in token request |
| **Old (without PKCE)** | No `code_verifier` sent, AI server proceeds without it (logs warning) |

### AI Server Log Messages

- **With PKCE:** `[Auth] Exchanging OAuth code for tokens (with PKCE)`
- **Without PKCE:** `[Auth] Exchanging OAuth code for tokens (without PKCE - legacy flow)`

**Note:** While backward compatible, it's recommended to update all desktop app instances to use PKCE for enhanced security.

## Security Considerations

### What PKCE Protects Against

| Attack | Protection |
|--------|------------|
| Authorization code interception | ✅ Protected |
| Malicious app on same device | ✅ Protected |
| Man-in-the-middle (with HTTPS) | ✅ Protected |

### What PKCE Does NOT Protect Against

| Attack | Mitigation |
|--------|------------|
| Compromised device/malware | Use secure token storage (keyring) |
| Phishing attacks | User education |
| Token theft after issuance | Short token expiry, refresh rotation |

### Additional Recommendations

1. **Token Storage:** Consider encrypting stored tokens using Windows Credential Manager (`keyring` library)
2. **Token Rotation:** Implement refresh token rotation when supported by the authorization server
3. **Monitoring:** Log authentication events for security monitoring
